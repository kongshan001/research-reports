## 📋 调研报告规范

### 目录管理

- 每个调研报告使用独立目录
- 目录命名使用 kebab-case（如 `skill-creator-research`）
- README 使用表格展示调研进度

### 落地过程要求 ⚠️

**必须包含本地部署验证测试案例**，包括：
- 环境检查命令和结果
- 功能验证测试步骤
- 实际输出截图/日志
- 遇到的问题和解决方案

### 基础信息

1. **背景需求** - 为什么需要
2. **目标** - 要解决什么问题
3. **设计方案** - 技术架构
4. **本地部署** - Win/Mac/Linux 部署指南
5. **效果展示** - 实际使用效果
6. **优缺点分析** - 优势与不足
7. **平替对比** - 对比其他类似的产品或工具
8. **落地过程** - 实践验证的完整记录（包括本地测试）

### 补充信息

9. **使用场景** - 具体什么情况下会用
10. **适合人群** - 开发者/团队/个人
11. **成本评估** - 免费还是付费、API 费用
12. **学习曲线** - 新手上手难度
13. **维护状态** - 是否在积极维护、最后更新时间
14. **社区活跃度** - Star 数、Issue 活跃度、贡献者数量
