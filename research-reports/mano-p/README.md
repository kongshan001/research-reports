## 背景需求

随着 AI Agent 技术的发展,计算机使用(Computer Use)场景的需求日益增长。企业和个人需要自动化 GUI 操作,但现有方案存在以下痛点:

- **数据安全问题**: 大部分方案(如 Anthropic Claude Computer Use、Manus)需要云端 API 调用,导致敏感数据离开本地设备
- **硬件依赖**: 传统方案依赖云端算力,响应延迟高,无法离线使用
- **平台限制**: 大多数方案仅支持 Web 应用,无法操作桌面软件、专业工具等
- **成本高昂**: 云端 API 调用按 token 收费,大规模自动化成本高

Mano-P 应运而生,旨在解决边缘设备上的本地化 GUI 自动化问题。

---

## 目标

Mano-P 是一个开源的**GUI-VLA(Vision-Language-Action)Agent 项目**,专为边缘设备设计:

1. **本地化运行**: 在 Mac mini/MacBook(M4 芯片 + 32GB RAM)上本地推理,数据不离开设备
2. **纯视觉交互**: 通过纯视觉理解实现跨平台 GUI 自动化,不限 Web 应用
3. **离线长任务**: 支持复杂业务流程的端到端自动化,无需互联网连接
4. **分阶段开源**: 逐步开源 Skills、模型 SDK、训练方法,满足不同开发者需求

---

## 设计方案

### 技术架构

Mano-P 构建于 [Mano Technical Report](https://arxiv.org/abs/2509.17336) 基础之上,核心技术包括:

- **Mano-Action 双向自强化学习**: Text↔Action 循环一致性学习,既能从描述定位元素,也能从元素生成描述
- **三阶段渐进训练**: SFT → 离线 RL → 在线 RL
- **"think-act-verify" 循环推理**: Think(思考) → Act(执行) → Verify(验证)的闭环推理机制
- **边缘优化**: 混合精度量化、视觉 token 剪枝、边缘推理适配

### 开源分阶段计划

| 阶段 | 开源内容 | 目标用户 |
|------|----------|----------|
| 第一阶段 | Mano-CUA Skills | Agent 爱好者(OpenClaw/Claude Code 用户) |
| 第二阶段 | 本地模型 + SDK | 高安全要求开发者 |
| 第三阶段 | 训练方法 + 量化技术 | 有定制训练需求的开发者 |

### 部署方式

1. **直接部署**: Mac mini/MacBook(M4 芯片 + 32GB+ RAM)
2. **算力棒部署**: 通过 USB 4.0+ 连接算力棒

## 优缺点分析

### 优势

1. **本地化运行**: 完全离线,数据不离开设备,高安全场景适用
2. **纯视觉交互**: 支持桌面软件、专业工具、3D 应用等非 Web GUI
3. **高性能推理**: 4B 量化模型达 476 tokens/s prefill,76 tokens/s decode
4. **端到端自动化**: 支持数十到数百步的复杂业务流程
5. **开源免费**: 分阶段开源,降低使用门槛

### 不足

1. **硬件限制**: 仅支持 Apple Silicon(M4 芯片),Windows/Linux 用户无法直接使用
2. **模型性能差距**: OSWorld 上比 Claude Sonnet 4.6(72.1%)低约 14%
3. **生态早期**: CLI 工具尚未发布,目前仅支持 OpenClaw/Claude Code Skill 集成
4. **算力棒成本**: 需要额外购买硬件设备

## 平替对比

| 特性 | Mano-P | OpenClaw | Claude Computer Use | Manus | 传统 RPA |
|------|--------|----------|---------------------|-------|-----------|
| 模型来源 | 本地内置模型 | 用户配置 | 云端 API | 云端 API | 无(规则引擎) |
| 数据安全 | ✅ 本地执行 | ⚠️ LLM/云调用 | ⚠️ 云端推理 | ⚠️ 云端推理 | ✅ 可本地 |
| 控制方式 | 纯视觉 | CDP+CLI | HTML 解析+CLI | HTML 解析+CLI | 系统 API |
| 使用场景 | 全平台 GUI | 跨平台应用 | 仅 Web 应用 | 仅 Web 应用 | 特定系统 |
| 长任务规划 | ✅ 自主规划 | ✅ 自主规划 | ✅ | ✅ | ❌ 需预设流程 |
| 响应速度 | ✅ 即时响应 | ✅ | ⚠️ 云延迟 | ⚠️ 云延迟 | ✅ 即时响应 |
| 部署成本 | ✅ 低门槛 | ✅ 免费开源 | ⚠️ 订阅费 | ⚠️ 订阅费 | ✅ 低成本 |
| 数据安全 | ✅ 本地执行 | ⚠️ 需配置 | ⚠️ 云端 | ⚠️ 云端 | ✅ 可本地 |

### 关键差异化

1. **vs OpenClaw**: 预置本地模型,开箱即用,无需用户配置
2. **vs Claude Computer Use**: 纯本地运行,离线可用,数据不离设备
3. **vs Manus**: 本地离线,纯视觉交互不限 Web
4. **vs 传统 RPA**: AI 自适应 UI 变化,无需预设工作流

## 使用场景

1. **企业业务自动化**: 复杂 GUI 操作的企业业务流程自动化
2. **跨系统数据集成**: 无 API 接口的多系统数据提取与整合
3. **智能报表生成**: 自动生成数据分析报告、工作总结等结构化文档
4. **高安全场景**: 敏感数据处理，离线环境部署
5. **个人效率工具**: 日常 GUI 操作自动化(测试、填报等)
6. **娱乐应用**: 游戏辅助(麻将等可视化游戏)
