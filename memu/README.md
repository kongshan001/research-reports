## 背景需求

当前 Agent 记忆系统的被动性问题：

1. **被动存储** — 记忆只在被查询时才被检索
2. **缺乏主动** — 不会主动分析、提炼、关联记忆
3. **人工触发** — 需要显式调用才能进行记忆操作
4. **碎片化** — 记忆之间缺乏主动关联和推理

memU 提出的核心理念：**让记忆本身变成一个 24/7 后台运行的主动 Agent**。

## 目标

memU 旨在实现**主动式记忆系统**：

- **记忆即 Agent** — 记忆系统本身是一个独立运行的 Agent
- **24/7 后台** — 持续分析、关联、提炼用户数据
- **主动学习** — 不等待查询，主动发现值得记住的信息
- **关系推理** — 主动关联不同记忆片段

## 设计方案

⚠️ **信息有限**：基于公开信息有限，以下为基于核心理念的架构推测。

### 核心架构推测

**1. 记忆 Agent 架构**

```
┌─────────────────────────────────────────┐
│           User Interaction              │
└─────────────────┬───────────────────────┘
                  │
┌─────────────────▼───────────────────────┐
│              Main Agent                  │
│  (处理用户请求、任务执行)                  │
└─────────────────┬───────────────────────┘
                  │ 双向通信
┌─────────────────▼───────────────────────┐
│           Memory Agent (memU)            │
│  ┌─────────────────────────────────────┐ │
│  │  24/7 后台运行                       │ │
│  │  - 主动分析用户对话                  │ │
│  │  - 主动提取关键信息                  │ │
│  │  - 主动关联记忆片段                  │ │
│  │  - 主动生成记忆摘要                  │ │
│  └─────────────────────────────────────┘ │
└─────────────────────────────────────────┘
```

**2. 主动记忆能力（推测）**

| 能力 | 说明 |
|------|------|
| 主动分析 | 不待查询，主动分析对话内容 |
| 主动提取 | 自动识别值得记忆的关键信息 |
| 主动关联 | 发现记忆间的潜在联系 |
| 主动摘要 | 自动生成记忆摘要和洞察 |
| 主动遗忘 | 识别并归档低价值记忆 |

**3. 记忆类型（推测）**

- **事实记忆** — 用户提供的明确信息
- **推断记忆** — 从行为中推断的隐含信息
- **关系记忆** — 记忆之间的关联
- **会话记忆** — 当前会话上下文

## 优缺点分析

### 优点

1. **主动性** — 不等待查询，主动分析
2. **关系推理** — 能发现记忆间的关联
3. **持续学习** — 24/7 后台持续改进
4. **智能摘要** — 自动生成记忆洞察
5. **范式创新** — 重新定义记忆系统角色

### 缺点

1. **资源消耗** — 24/7 运行资源占用高
2. **复杂度** — 系统架构复杂
3. **信息有限** — 公开资料少，学习难度大
4. **成本高** — 持续运行 LLM 调用成本
5. **可控性** — 主动行为可能产生意外结果
6. **隐私考量** — 持续监控分析隐私风险

## 平替对比

| 框架 | 主动性 | 架构 | 复杂度 | 适合场景 |
