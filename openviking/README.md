## 背景需求

Agent 记忆系统的**个性化不足**问题：

1. **通用化** — 大多数框架面向通用场景
2. **缺乏人格** — 记忆与 Agent 人格分离
3. **上下文浅** — 只记住事实，缺乏深层理解

OpenViking 旨在构建**具有深层人格的个性化记忆系统**。

## 目标

OpenViking 旨在实现**人格化记忆系统**：

- **人格一体化** — 记忆与 Agent 人格深度绑定
- **情感记忆** — 记住情感上下文，不只是事实
- **叙事构建** — 将记忆组织成连贯的个人叙事

## 优缺点分析

### 优点

1. **人格化** — 记忆与 Agent 人格深度绑定
2. **情感记忆** — 记住情感上下文
3. **叙事组织** — 将记忆组织成连贯故事
4. **创新性强** — 不同于传统记忆系统

### 缺点

1. **信息有限** — 公开资料极少
2. **复杂度高** — 人格模型设计复杂
3. **难以评估** — 人格化难以量化
4. **资源消耗** — 多层记忆模型资源占用大

## 平替对比

| 框架 | 核心特点 | 复杂度 | 适合场景 |
