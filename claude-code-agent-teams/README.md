# Claude Code Agent Teams 调研

## 1. 背景需求

单一大语言模型在处理复杂任务时存在局限：
- **串行执行效率低** - 多任务需要依次处理
- **上下文容量有限** - 单一模型无法同时深入多个领域
- **缺乏协作机制** - 多代理间无法直接沟通协调
- **无法并行探索** - 无法同时从不同角度分析问题

## 2. 目标

Claude Code Agent Teams 是 **多代理协作系统**，让多个 Claude Code 实例协同工作：
- 一个会话担任团队负责人，协调工作、分配任务
- 队友各自独立工作，拥有独立上下文窗口
- 队友之间可以直接沟通，共享任务列表
- 支持并行探索和交叉验证

## 3. 设计方案

### 核心架构

```
┌─────────────────────────────────────────────────────────────┐
│                    Agent Teams 架构                           │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────┐                                          │
│   │   Team Lead  │ ←── 协调者，分配任务，整合结果              │
│   └──────┬───────┘                                          │
│          │                                                   │
│    ┌─────┴─────┬─────────────────┐                         │
│    ▼           ▼                 ▼                          │
│ ┌──────┐   ┌──────┐          ┌──────┐                      │
│ │ Teammate │  │ Teammate │  │ Teammate │  ←── 独立 Claude  │
│ │ (UX)    │  │ (Tech)   │  │ (Review) │     实例          │
│ └──────┘   └──────┘          └──────┘                      │
│                                                             │
│   共享任务列表 ←── 任务分配、claim、完成状态                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 与 Subagents 对比

| 特性 | Subagents | Agent Teams |
|------|-----------|-------------|
| **上下文** | 独立上下文，结果返回主代理 | 独立上下文，完全独立 |
| **沟通** | 只报告结果给主代理 | 队友间直接消息 |
| **协调** | 主代理管理所有工作 | 共享任务列表，自协调 |
| **适用** | 专注任务，只关心结果 | 复杂工作，需讨论协作 |
| **Token 成本** | 较低 | 较高 |

### 核心功能

| 功能 | 说明 |
|------|------|
| **团队创建** | 自然语言描述任务和团队结构 |
| **任务分配** | 显式分配或队友自claim |
| **并行执行** | 多任务同时处理 |
| **直接沟通** | 可与任意队友直接对话 |
| **计划审批** | 复杂任务需审批后才能执行 |
| **优雅关闭** | 请求关闭，队友可拒绝 |
| **质量门禁** | Hooks 强制质量规则 |

### 显示模式

1. **In-process**：所有队友在主终端内运行
   - 使用 Shift+Down 切换队友
   - 无需额外配置

2. **Split panes**：每个队友独立面板
   - 需要 tmux 或 iTerm2
   - 可同时查看所有队友输出

## 4. 本地部署

### 前置要求

- Claude Code 已安装
- 实验性功能需手动启用

### 启用 Agent Teams

在 `settings.json` 中添加：

```json
{
  "env": {
    "CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS": "1"
  }
}
```

或设置环境变量：

```bash
export CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1
```

### 使用方式

```bash
# 创建团队
创建一个代理团队，从不同角度探索这个问题：
一个关注UX，一个关注技术架构，一个扮演反对派

# 分配任务
让 researcher 队友负责调研市场
让 architect 队友负责设计架构

# 审批计划
让 architect 队友规划认证模块重构，需审批后才能执行

# 关闭队友
让 researcher 队友关闭

# 清理团队
清理团队
```

## 6. 优缺点分析

### ✅ 优点

| 优点 | 说明 |
|------|------|
| **并行效率** | 多任务同时处理 |
| **多角度分析** | 不同队友探索不同方向 |
| **直接沟通** | 队友间可直接交流 |
| **独立上下文** | 每个队友有完整上下文 |
| **灵活协调** | 支持自协调和显式分配 |

### ❌ 缺点

| 缺点 | 说明 |
|------|------|
| **实验性** | 默认禁用，有已知限制 |
| **Token 成本高** | 每个队友独立实例 |
| **协调开销** | 需要管理团队复杂性 |
| **依赖配置** | 需要 tmux/iTerm2 才能用分屏 |
